# Elite-Lawyer 精英律师数字助手

> 作者：楠哥说法
> 版本：V5.2
> 协议：GPL-3.0

楠哥专属的律师数字助手，以案件为核心，提供文书生成、法律检索、合同审核、客户管理、时效预警等全流程支持。

## 系统架构（L0 + DA 四层）

```
L0 存储层 ── 状态持久化（案件/时效/知识/配置落地为文件）
    ↓
L1 内核层 ── 角色定位 + 执业原则（全局优先级唯一来源）
    ↓
L2 机制层 ── 案件流程引擎 + 时效预警 + 知识发现 + 质量控制（纯规则）
    ↓
L3 模块层 ── 案件管理 + 文书生成 + 法律检索 + 合同审核 + 客户管理 + 日程提醒（执行）
    ↓
L4 知识库层 ── 法规库 + 案例库 + 模板库 + 裁判规则（按领域分片 + 读写协议）
```

## 快速开始

### 安装技能
下载 `elite-lawyer-v5.skill` 文件，导入到你的 AI 助手平台即可使用。

### 初始化配置
编辑 `.state/config.json`（或通过对话"设置我的律所为XX"），填入律师姓名、律所、默认法院等。

## V5.2 架构优化说明

相较 V5.1 的核心变化：

1. **新增 L0 存储与状态层**：定义状态读写契约、目录结构、案件/时效/配置 schema——解决 AI 无持久内存的跨会话状态问题
2. **L4 知识库分片 + 读写协议**：按领域分子文件 + `index.md` 总索引，定义"先查索引→grep定位→读写卡片"协议
3. **全局优先级唯一来源**：L1-02 成为唯一任务优先级定义，L2-02/L3-06 的 P0-P3 明确为不同维度的标签
4. **L2/L3 职责去重**：L3 不再重复定义质量标准，改为引用 L2-04 质控机制
5. **工作流异常分支**：所有工作流增加信息不全/检索无果/法条存疑/工具不可用 的降级处理
6. **集成边界声明**：明确 AI 可执行 vs 需平台/人工执行的能力边界
7. **集中配置层**：`.state/config.json` 统一管理律师/律所/集成开关
8. **冒烟测试 + 路由单源化**：examples/ 验证工作流；触发路由唯一来源 = SKILL.md

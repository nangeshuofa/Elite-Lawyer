# Elite-Lawyer 精英律师数字助手

> 作者：楠哥说法
> 版本：V6.1
> 协议：GPL-3.0

楠哥专属的律师数字助手，以案件为核心，提供文书生成、法律检索、合同审核、客户管理、时效预警等全流程支持。

> **V6.1 架构原则**：适应业务变化 · 支持持续演进 · 在稳定性/可维护性/可扩展性之间取得平衡（稳定性 > 可维护性 > 可扩展性）。

## 系统架构（L0 + DA 四层 + 扩展点）

```
L0 存储层 ── 状态持久化 + 架构约束 + 扩展点（模块注册表/领域配置/演进指南/集成接口）
    ↓
L1 内核层 ── 角色定位 + 执业原则（全局优先级唯一来源）
    ↓
L2 机制层 ── 案件流程引擎 + 时效预警 + 知识发现 + 质量控制（纯规则）
    ↓
L3 模块层 ── 案件管理 + 文书生成 + 法律检索 + 合同审核 + 客户管理 + 日程提醒（可扩展）
    ↓
L4 知识库层 ── 法规库 + 案例库 + 模板库 + 裁判规则（按领域分片 + 读写协议，可扩展）
```

## 快速开始

### 安装技能
下载 `elite-lawyer-v5.skill` 文件，导入到你的 AI 助手平台即可使用。

### 初始化配置
编辑 `.state/config.json`（或通过对话"设置我的律所为XX"），填入律师姓名、律所、默认法院、启用领域。

## V6.0 架构升级说明

相较 V5.2 的核心变化（围绕"适应变化·持续演进·三性平衡"）：

### 1. 适应业务变化（可扩展性）
- **模块注册表**：新增功能模块只需登记 + 创建文件，不改核心逻辑
- **领域配置**：业务领域（医疗/劳动/合同/知产/刑事...）可配置启用，知识库按领域适配
- **扩展点设计**：功能模块/业务领域/知识内容/时效类型/模板/配置 均有明确定义

### 2. 支持持续演进（版本与兼容性）
- **schema_version**：存储结构带版本号（当前 v2），变更可追溯
- **演进指南**：版本升级、数据迁移、扩展能力的标准流程
- **兼容性承诺**：旧版 `.state/` 数据自动兼容，缺失字段补默认值
- **语义化版本**：主/次/修 三级，破坏性变更有迁移窗口

### 3. 三性平衡（稳定性 > 可维护性 > 可扩展性）
- **架构约束**：单向依赖、单一职责（Single Source of Truth）、禁止循环依赖
- **稳定性护栏**：输出可信度分级（✅已核验 / ⚠️未核验 / ❓待确认）强制标注，针对法律准确性软肋
- **空库不编造**：知识库无结果时提示而非编造法条
- **可维护性**：依赖方向清晰、扩展有规范、变更有检查清单

## V6.1 外部集成预留说明

相较 V6.0 的变化（围绕"适应业务变化·持续演进"，`schema_version` 仍为 2，向后兼容）：

### 1. 外部集成扩展点（新增）
- **集成接口.md（L0）**：定义两个**预留**外部集成扩展点，默认 `enabled=false`，不影响现有行为
  - **I01 北大法宝法律数据库 MCP**：外部权威知识源，本地 L4 无果或需实时核验时由 `L2 知识发现机制` 路由调用，结果标 ✅ 已核验，失败回退本地 + ❓ 待确认
  - **I02 腾讯 ima 接口**：外部客户端 / 知识同步通道，以本地 `.state/` + `L4` 为唯一真相源，冲突以本地为准
- **config.json 预留字段**：`mcp_sources.pkulaw` / `integrations.tencent_ima`（默认关闭，向后兼容）
- **架构约束扩展点表**新增"外部集成"类别

### 2. 领域术语修正
- 法规库分片 `医疗法.md` 更名为 `医事法.md`，相关引用（领域配置 / index / 01_法规库 / 知识发现机制）同步更新

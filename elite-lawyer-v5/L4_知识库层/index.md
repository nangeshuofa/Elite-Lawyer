> 本文件是知识库的总入口。AI 检索知识时**先读本索引定位文件**，再深入对应分片文件。写入知识时**同步更新本索引**。

## 二、检索协议（AI 读取知识）

```
Step 1 — 读 index.md，根据领域+类型定位分片文件
Step 2 — 用 Grep 工具在目标文件检索关键词（案由/法条号/争议点）
Step 3 — 命中卡片 → 返回卡片全文 + 标注来源文件
Step 4 — 无命中 → 明确告知"知识库未收录"，附检索范围，不编造
```

## 三、写入协议（AI 沉淀知识）

```
Step 1 — 判断知识类型（RULE/TPL/CASE/STRAT/KNOW）
Step 2 — 定位目标分片文件（按领域+类型）
Step 3 — 用 Edit 工具在文件末尾追加标准格式卡片
Step 4 — 同步更新本 index.md（如新增了领域标签）
Step 5 — 标注写入时间 + 来源
```

## 四、知识卡片标准格式

```markdown
### [类型] 知识标题
- **来源**：案件名/任务名/日期
- **领域**：医疗/劳动/合同/...
- **摘要**：一句话核心结论
- **详情**：具体内容（3-5句，可引原文）
- **法律依据**：相关法条/司法解释
- **复用场景**：何时可再次使用
- **关联**：相关案例/模板/文件路径
- **添加日期**：YYYY-MM-DD
```

## 六、维护规则

1. **索引先行**：新增知识前先确认 index.md 是否已覆盖该领域，未覆盖则新增分片文件并登记
2. **去重合并**：写入前 grep 查重，相似内容合并而非新增
3. **来源可查**：每条知识必须标注来源和日期
4. **定期健康**：每季度检查失效法条标记、重复合并（见 L2-03 知识发现机制）

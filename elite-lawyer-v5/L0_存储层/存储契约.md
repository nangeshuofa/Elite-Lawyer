# L0 存储与状态层 — 状态读写契约

> 本层是技能的"持久化根基"。AI Agent 本身无持久内存，所有跨会话状态（案件、时效、知识、配置）必须落地为文件，由 AI 通过文件编辑工具读写。
> **Schema 版本**：`schema_version: 2`（V6.0 起）。旧版数据迁移见第六节。

## 一、目录结构

```
elite-lawyer-v5/
├── .state/                    # 运行时状态（AI 通过文件工具读写）
│   ├── cases/                 # 每个案件一个目录，以案件编号命名
│   │   └── CASE-2026-MED-001/
│   │       ├── meta.json      # 案件元数据（含 schema_version）
│   │       ├── timeline.json  # 时效节点 + 预警状态
│   │       └── evidence.md    # 证据清单
│   ├── deadlines.json         # 全局时效总表（供周一扫描）
│   └── config.json            # 配置（律师/律所/集成开关/领域/扩展点）
├── L4_知识库层/               # 知识物理存储
│   ├── index.md               # 知识索引
│   ├── 法规库/ 案例库/ 模板库/ 裁判规则/  # 按领域分片
└── SKILL.md
```

## 二、状态读写契约

### 会话启动协议（每次新会话）
```
1. 读取 .state/config.json → 恢复配置（律师名、律所、默认法院等）
2. 读取 .state/deadlines.json → 恢复时效总表，检查有无今日/近期到期
3. 如用户提及某案件 → 读取 .state/cases/<案件编号>/meta.json 恢复案件上下文
4. 如涉及知识检索 → 读取 L4_知识库层/index.md 定位文件
```

### 状态写入协议
| 触发动作 | 写入位置 | 格式 |
|----------|----------|------|
| 新建案件 | `.state/cases/<编号>/meta.json` | JSON（见下方 schema） |
| 录入/更新时效 | `.state/deadlines.json` | JSON 数组追加/更新 |
| 证据变更 | `.state/cases/<编号>/evidence.md` | Markdown 表格 |
| 配置变更 | `.state/config.json` | JSON |
| 知识沉淀 | `L4_知识库层/<库>/<领域>.md` | 标准卡片 + 同步 index.md |

### 文件写入原则
- **即时写入**：状态变更后立刻写回，不依赖会话结束
- **原子更新**：修改 JSON 用整体替换，避免部分写入
- **路径规范**：案件编号格式 `年份-领域-序号`（如 `2026-MED-001`）
- **不删历史**：知识库更新用追加/修订，废止内容标记而非删除

## 四、存储边界

| 存哪里 | 不存哪里 |
|--------|----------|
| 案件结构化数据（.state/cases） | 已生成的文书最终稿（归案件物理目录，非 .state） |
| 时效总表（.state/deadlines.json） | 客户闲聊记录 |
| 配置（.state/config.json） | 一次性问答上下文 |
| 知识卡片（L4 物理文件） | 临时检索结果 |

> 设计原则：`.state/` 只存"跨会话必须恢复的最小状态"，完整卷宗按 L2-01 案件文件目录标准存放于案件工作区。

## 五、Schema 版本与兼容性

> 本契约随技能版本演进。`schema_version` 字段标记数据结构版本，确保旧数据在新版技能下不失效（持续演进原则）。

### 版本记录
| schema_version | 引入版本 | 变更内容 |
|----------------|----------|----------|
| 1 | V5.2 | 初始 schema：meta/timeline/deadlines/config 基础结构 |
| 2 | V6.0 | config.json 增加 `schema_version`/`domains`/`extensions` 字段；meta.json 增加 `schema_version` 字段；新增领域配置与模块注册表引用 |

### 向后兼容承诺
- **旧数据不失效**：V6.0 读取 V5.2 的 `.state/` 数据时，缺失的新字段使用默认值，不报错
- **新增字段可选**：config.json 的 domains/extensions 缺失时，技能回退到"内置默认领域"运行
- **不破坏已有案件**：案件 meta.json 无 schema_version 时，视为 v1，正常读取

### 迁移规则
| 场景 | 处理方式 |
|------|----------|
| V5.2 升级到 V6.0 | 自动：读取旧数据→补默认字段→无需人工迁移 |
| 未来 V6.x 小版本 | 字段增量追加，旧数据自动兼容 |
| 跨大版本（V7+） | 提供迁移脚本说明（见 `演进指南.md`）|

### schema_version 写入规范
- 所有 `.state/` 下的 JSON 文件必须包含 `"schema_version"` 字段
- 当前值：`2`
- 升级 schema 时：旧数据读取方负责兼容，写入方使用新版号

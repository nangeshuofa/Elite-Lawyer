# 集成接口 — 外部系统扩展点（预留）

> 本文件定义技能对**外部系统**的扩展点契约。当前 V6.1 为"**预留（reserved）**"阶段：接口规范已定义、配置字段已保留，但默认**未启用、未实现**具体适配器。
> 设计目标：**适应业务变化**——通过标准接口接入专业法律数据库（北大法宝）与协作客户端（腾讯 ima），无需改动核心逻辑。
> 所有外部集成必须遵循 `架构约束.md` 三性平衡与 `SKILL.md` 输出可信度分级（稳定性护栏）。

---

## 一、外部集成总览

| 接口ID | 名称 | 类型 | 状态 | 用途 |
|--------|------|------|------|------|
| **I01** | 北大法宝法律数据库 MCP | 外部知识源（legal_db_mcp） | 预留（reserved） | 实时检索法规/案例/裁判规则，作为 L4 知识库的外部权威补充 |
| **I02** | 腾讯 ima 接口 | 外部客户端 / 知识同步（knowledge_client） | 预留（reserved） | 知识库同步、跨端调用、作为替代对话入口 |

> **激活条件**：在 `.state/config.json` 将对应字段 `enabled` 置为 `true` 并填充凭证后，由**平台/研发**实现适配器（见各接口"激活流程"）。reserved 状态下技能行为与 V6.0 完全一致，不影响稳定性。

---

## 二、I01 北大法宝法律数据库 MCP（外部知识源）

### 定位
- 作为 **L4 知识库层的外部权威补充源**。当本地 L4 知识库无结果或需实时核验法条/案例有效性时，由 `L2_机制层/03_知识发现机制.md` 的"外部源路由"路径转发到此 MCP。
- 与本地 L4 的关系：**本地为主、MCP 为辅**。MCP 结果用于实时核验与扩源，不替代本地沉淀的标杆案例与律师经验。

### 接口契约（预留）

| 项 | 规范 |
|----|------|
| 接口类型 | MCP（Model Context Protocol）Server |
| 数据源 | 北大法宝（pkulaw）法规库 / 案例库 / 裁判规则库 |
| 认证方式 | MCP Server 鉴权（token / 平台凭据，存于 `config.json`，**不硬编码**） |
| 查询适配 | 关键词 / 法条号 / 案号 / 裁判规则 → MCP 检索原语 |
| 返回结构 | 标准知识卡片（对齐 L4 知识分类体系 `[RULE]`/`[CASE]`/`[TPL]`/`[STRAT]`/`[KNOW]`） |
| 可信度 | MCP 实时权威结果 → ✅ 已核验；网络/鉴权失败 → 降级 ⚠️ 或 ❓ |
| 失败处理 | 不可用时**回退本地 L4** + 标注 ❓ 待确认，**严禁编造**法条或案号 |

### 激活流程（实现时由平台/研发执行）
```
Step 1 — config.json 的 mcp_sources.pkulaw.enabled = true，填充凭据
Step 2 — 平台部署/接入对应的 MCP Server 适配器（遵循本契约查询/返回结构）
Step 3 — L2 知识发现机制「外部源路由」启用：本地无果 → 调用 MCP
Step 4 — 冒烟测试覆盖：检索无果 / 鉴权失败 / 网络超时 三种降级场景
```

## 三、I02 腾讯 ima 接口（外部客户端 / 知识同步）

### 定位
- 作为**外部客户端与知识同步通道**。ima 可作为技能知识库的同步目标/来源，或作为替代对话入口。
- 与本地 L0/L4 的关系：知识以本地 `.state/` + `L4_知识库层/` 为**唯一真相源（Single Source of Truth）**；ima 同步为桥接层，冲突以本地为准。

### 接口契约（预留）

| 项 | 规范 |
|----|------|
| 接口类型 | HTTP / SDK 客户端接口 |
| 数据源 | 腾讯 ima 个人/团队知识库 |
| 认证方式 | ima 开放平台凭据（存于 `config.json`，**不硬编码**） |
| 同步方向 | 知识库同步（导入 / 导出）、跨端调用 |
| 同步协议 | 增量同步 + 版本标记，避免覆盖本地已审核知识 |
| 冲突策略 | 本地 `.state/` + `L4` 为权威；ima 内容标记为"外部来源" |
| 失败处理 | 同步失败**不阻塞主流程**，记录待重试并标注 ⚠️ |

### 激活流程（实现时由平台/研发执行）
```
Step 1 — config.json 的 integrations.tencent_ima.enabled = true，填充凭据
Step 2 — 实现 ima 同步适配器（导入/导出/冲突处理）
Step 3 — 配置同步频率与方向（默认：本地 → ima 单向导出）
Step 4 — 冒烟测试覆盖：首次同步 / 增量同步 / 冲突场景
```

## 四、外部集成通用约束（稳定性护栏）

1. **不硬编码凭据**：所有 token / 凭据仅在 `config.json` 配置，禁止写入逻辑文件
2. **不破坏本地真相源**：外部同步以本地 `.state/` + `L4` 为权威，冲突以本地为准
3. **可信度必标注**：外部源输出必须带 ✅ / ⚠️ / ❓ 等级，不可用则 ❓ 待确认
4. **失败可降级**：任一外部集成不可用，必须回退本地能力并声明局限，**不得静默失败或编造**
5. **默认关闭**：reserved 状态默认 `enabled=false`，不影响现有行为（向后兼容）
6. **扩展不破坏核心**：新增外部集成仅通过本文件登记，不改 L1–L4 核心逻辑

> 本文件为"预留"规范：当前所有接口 `enabled=false`，技能行为与 V6.0 完全一致，仅为后续接入专业法律数据库与协作客户端预留标准扩展点。

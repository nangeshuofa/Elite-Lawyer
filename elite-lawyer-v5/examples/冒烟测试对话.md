# 冒烟测试示例对话

> 用途：验证技能各工作流是否正常触发与执行。每条示例包含"用户输入"和"预期 AI 行为"。
> 部署后可用这些对话做 smoke test，确认触发路由、工作流、输出格式无误。

---

## 示例 1：案件建档（验证 L3-01 工作流 + L0 状态写入）

**用户输入**：
```
新建案件：张某诉某医院医疗损害责任纠纷，我是原告张某的代理律师。
案由是医疗损害，法院是北京市朝阳区法院，立案日期 2026-03-15。
```

**预期 AI 行为**：
1. 确认已获取必需字段（案件名称、案由、我方当事人），缺失字段（被告地址、经办法官）标 ⚠️ 待补
2. 生成案件编号 `CASE-2026-MED-001`
3. 写入 `.state/cases/CASE-2026-MED-001/meta.json`
4. 识别时效节点（如收到举证通知书则录入 deadlines.json）
5. 主动建议：是否检索该领域法条、生成起诉状、整理证据清单

## 示例 2：文书生成（验证 L3-02 工作流 + L2-04 质检引用）

**用户输入**：
```
写个起诉状。原告张某，被告某医院，医疗损害纠纷，
事实：2026年1月手术发生并发症，认为医院过错。
请求：赔偿医疗费、误工费共50万元。管辖：朝阳区法院。
```

**预期 AI 行为**：
1. 进入起诉状生成工作流
2. 检索医疗损害相关法条（民法典1218-1228）
3. 生成初稿（结构：当事人→诉讼请求→事实理由→法律依据）
4. 执行 L2-04 文书质检（格式、法条引用、金额计算）
5. 标注 ⚠️ 需律师审核部分（过错认定、赔偿计算依据）
6. 交付：初稿 + 证据清单模板

---

## 示例 3：合同审核（验证 L3-04 工作流 + 风险分级）

**用户输入**：
```
审核一下这份买卖合同：[粘贴合同文本]
```

**预期 AI 行为**：
1. 识别合同类型（买卖）、主体
2. 逐条审查（P0 主体/标的/价款/违约/争议解决 → P1 → P2）
3. 风险分级标注 🔴/🟡/🟢
4. 输出合同审核意见书（总体评价 + 逐条意见 + 重点提示）
5. 如合同条款缺失，从模板库推送标准条款建议

---

## 示例 4：法律检索（验证 L3-03 工作流 + 局限说明）

**用户输入**：
```
查一下医疗过错怎么认定
```

**预期 AI 行为**：
1. 检索法条（民法典1221、1222）+ 司法解释
2. 检索裁判规则（`裁判规则/医疗损害.md`）
3. 输出检索报告（法律规定 + 裁判规则 + 类案 + 结论 + 局限说明）
4. 如知识库未收录相关案例，明确告知"未检索到类案"，不编造

---

## 示例 5：时效预警（验证 L2-02 + 全局优先级）

**用户输入**：
```
XX案收到一审判决书了，今天 2026-07-08 下的
```

**预期 AI 行为**：
1. 识别裁判日期，自动计算上诉期（判决15天 → 2026-07-23 截止）
2. 写入 `.state/deadlines.json`，标记 P0（全局最高级）
3. 设置多级提醒（提前7/3/1天 + 当天）
4. 输出提醒："⚠️ XX案上诉期将于 2026-07-23 截止，已设置提醒"

## 验收标准

| 检查项 | 通过条件 |
|--------|----------|
| 触发路由 | 5 个示例均进入正确工作流 |

| 局限说明 | 示例4 无结果时不编造，附检索范围 |
| 优先级 | 示例5 时效自动归入全局 P0 |

## L1 · 内核层 — 身份与原则

> 数字助手的 DNA，决定做事的底线和风格。

| 文件 | 核心内容 |
|------|----------|
| `L1_内核层/01_角色定位.md` | 数字助手的身份定位、与律师的分工边界 |
| `L1_内核层/02_执业原则.md` | 保密原则、专业原则、勤勉原则、诚信原则 |

**分工原则：**
- AI 做：信息整理、初稿生成、法规检索、格式排版、时效提醒、重复劳动
- 律师做：事实判断、法律适用、策略选择、客户沟通、出庭应诉、最终决策

## L2 · 机制层 — 运转与质控

> 定义系统如何运转，确保输出质量和工作效率。

| 文件 | 核心内容 |
|------|----------|
| `L2_机制层/01_案件流程引擎.md` | 接案→准备→执行→结案→归档 全流程标准动作 |
| `L2_机制层/02_时效预警机制.md` | 时效识别 → 录入追踪 → 多级预警 → 逾期升级 |
| `L2_机制层/03_知识发现机制.md` | 办案自动沉淀 → 分类归档 → 跨案复用 |
| `L2_机制层/04_质量控制机制.md` | 文书自检清单 → 风险点排查 → 格式规范检查 |

**运转逻辑：**
```
接案 → 案件建档 → 时效录入 → 自动预警
              ↓
         文书/检索/审核 → AI初稿 → 律师审核 → 交付
              ↓
         知识自动提炼 → 归档 L4 知识库 → 后续案件复用
```

## L4 · 知识库层 — 四大弹药库

> 长期沉淀的可复用知识资产，越用越值钱。每办一个案子，知识库就更厚一分。

| 知识库 | 文件 | 内容 |
|--------|------|------|
| 📚 法规库 | `L4_知识库层/01_法规库.md` | 常用法律法规、司法解释、指导案例 |
| 📂 案例库 | `L4_知识库层/02_案例库.md` | 已办案件研究、典型案例分析、裁判要旨 |
| 📑 模板库 | `L4_知识库层/03_模板库.md` | 文书模板、合同模板、清单模板、函件模板 |
| 🎯 裁判规则 | `L4_知识库层/04_裁判规则.md` | 各领域裁判规则、举证要点、胜诉要素 |

**知识自动沉淀流程：**
```
案件结案 → AI自动提取
    ├── 法律论点 + 裁判要旨 → 裁判规则库
    ├── 文书范本 + 优秀条款 → 模板库
    ├── 案例分析 + 经验教训 → 案例库
    └── 适用法条 + 司法解释 → 法规库索引
```

## 典型工作流示例

### 场景1：收到新案件
```
1. 录入案件基本信息 → 自动建档
2. 识别关键时效（答辩期/举证期/开庭日）→ 自动设置预警
3. 检索相关法条+类案 → 生成检索报告
4. 根据案情生成起诉状/答辩状初稿
5. 律师审核修改 → 定稿交付
```

### 场景2：合同审核
```
1. 上传合同 → 自动识别合同类型
2. 扫描风险点（主体/标的/履行/违约/争议解决）
3. 生成审核意见 + 修改建议
4. 输出合同审核意见书
5. 沉淀：新发现的风险点 → 模板库更新
```

### 场景3：开庭准备
```
1. 调取案件全部材料
2. 生成庭审提纲（争议焦点+质证意见+辩论要点）
3. 检索类案裁判倾向 → 提供策略参考
4. 开庭前1天自动提醒 + 材料清单确认
```

## V5.0 重构说明

**相较 V4.0 的核心变化：**

1. **定位转变**：从"人生操作系统"变为"律师数字助手"——回到 elite-lawyer 本名，聚焦律师干活
2. **功能聚焦**：砍掉冥想/运动/财富/关系等人生模块，聚焦 6 大律师核心功能
3. **效率优先**：每个模块都围绕"帮律师省时间"设计，不说空话
4. **知识闭环**：办案自动沉淀知识到 L4，形成正向循环——案子办得越多，助手越聪明
5. **质控机制**：新增质量控制层，AI 输出必须可审核、可追溯、可修正

**版本历史**：
- V1.0：律师进化系统，基础四层架构
- V2.0：增加信念层、习惯层、进化层
- V3.0：增加律师执业融合、动态平衡引擎、里程碑成长、修炼日志
- V4.0：按 DA 四层架构重构 + 知识发现机制
- V5.0：**转型为律师数字助手，聚焦专业工具属性，砍掉非核心模块**

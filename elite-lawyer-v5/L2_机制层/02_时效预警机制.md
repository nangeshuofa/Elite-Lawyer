# 时效预警机制 — 零遗漏保障

> 核心原则：时效是律师的生命线，漏了就是事故。机器比人靠谱，全部由系统追踪。

## 时效类型与等级

> **优先级说明**：本机制的 P0/P1/P2 指"时效自身的紧急程度"（决定提前提醒天数），与全局任务优先级（见 `L1_内核层/02_执业原则.md` 第五章）是不同维度。任何临近的时效节点在全局任务排序中自动归入 P0 最高级。

### 时效等级 P0 — 绝对不能漏（红色预警）
| 时效类型 | 时限 | 提前提醒 |
|----------|------|----------|
| 上诉期（判决） | 15天 | 7天、3天、1天、当天 |
| 上诉期（裁定） | 10天 | 7天、3天、1天、当天 |
| 举证期限 | 法院指定 | 7天、3天、1天、当天 |
| 答辩期 | 15天 | 7天、3天、1天、当天 |
| 开庭日 | 法院指定 | 3天、1天、当天早上 |

### 时效等级 P1 — 重要（黄色预警）
| 时效类型 | 提前提醒 |
|----------|----------|
| 申请执行期限（2年） | 6个月、1个月、1周 |
| 再审申请期限（6个月） | 2个月、2周、3天 |
| 保全期限（查封/冻结） | 到期前 30天、7天 |
| 合同到期续约提醒 | 60天、30天、7天 |

### 时效等级 P2 — 常规（蓝色提醒）
| 时效类型 | 提前提醒 |
|----------|----------|
| 顾问合同到期 | 30天、15天、7天 |
| 律师执业证年审 | 1个月、1周 |
| 年度考核 | 1个月 |
| 培训学分 | 每季度提醒 |

## 时效录入

### 自动识别
- 收到判决/裁定 → 自动识别裁判日期 → 自动计算上诉期
- 收到举证通知书 → 自动识别举证期限
- 收到开庭传票 → 自动录入开庭时间

### 手动录入
- 合同时效：签约时录入合同到期日、付款日等
- 客户约定：XX日前提交XX文件
- 其他时效：任何有明确截止日期的事项

### 录入字段
```
时效名称：___________
时效类型：___________
截止日期：___________
关联案件：___________
关联客户：___________
优先级：P0/P1/P2
备注：___________
```

## 多级预警机制

```
┌─────────────────────────────────────────────────┐
│              时效倒计时                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━  剩余 3 天              │
└─────────────────────────────────────────────────┘
         ↓              ↓              ↓
      提前7天         提前3天         提前1天
      蓝色提醒        黄色提醒        红色预警
    （常规通知）    （重点关注）    （必须确认）
         ↓              ↓              ↓
      飞书消息      飞书消息+      飞书消息+
                              邮件备份     邮件+待办置顶
```

### 预警分级
| 级别 | 触发条件 | 通知方式 | 确认要求 |
|------|----------|----------|----------|
| 🔵 蓝色提醒 | 提前7天以上 | 飞书消息 | 无需确认 |
| 🟡 黄色预警 | 提前3天 | 飞书消息 + 邮件备份 | 建议确认 |
| 🔴 红色预警 | 提前1天/当天 | 飞书消息 + 邮件 + 待办置顶 | 必须确认收到 |
| ⚫ 黑色警报 | 已逾期 | 立即通知 + 紧急补救方案 | 立即处理 |

## 漏时效应急机制

> 时效事故 = 执业事故。一旦发生，立即启动以下 SOP，以最小化损失为第一目标。

### 应急响应 SOP

#### 第一阶段：立即止损（0-30分钟内）
| 步骤 | 动作 | 负责人 |
|------|------|--------|
| 1 | 立即通知主办律师，说明时效状态 | AI 自动 + 人工 |
| 2 | 确认精确的剩余时间（精确到小时） | AI |
| 3 | 评估是否还有补救窗口（如邮寄以邮戳为准、电子提交以系统时间为准） | 律师判断 |
| 4 | 如果当天截止 → 立即启动加急处理 | 律师 + 助手 |

#### 第二阶段：评估补救方案（30分钟-2小时）
按时效类型评估补救可能：

**上诉期逾期：**
- 检查是否有法定顺延事由（不可抗力、其他正当理由）
- 评估是否可以申请恢复上诉期（《民事诉讼法》第86条）
- 如果可以，立即准备恢复上诉期申请材料
- 如果不可以 → 启动替代方案（再审、抗诉等）

**举证期逾期：**
- 立即联系经办法官/书记员说明情况
- 申请延长举证期限（需有正当理由）
- 准备延期申请材料 + 已完成部分的证据
- 评估逾期举证的后果（失权/训诫/罚款）

**答辩期逾期：**
- 立即提交答辩状（即使不完整也比不交好）
- 联系法院说明情况
- 庭前会议/庭审中补充答辩意见

**申请执行期限逾期：**
- 检查是否有时效中断/中止事由
- 确认最后履行日期（执行时效从履行期届满起算）
- 如果有中断事由 → 整理证据材料立即申请

**保全期限到期：**
- 立即联系法院确认是否需要续封/续冻
- 准备续保申请材料
- 与当事人确认是否继续保全

#### 第三阶段：风险对冲（同步进行）
| 风险 | 对冲措施 |
|------|----------|
| 执业责任风险 | 检查执业责任险是否覆盖此类事故 |
| 客户索赔风险 | 主动与客户沟通，如实说明情况，提出补救方案 |
| 法院处罚风险 | 主动配合，诚恳说明原因，争取从轻处理 |
| 声誉风险 | 内部复盘，不对外扩散；吸取教训，优化流程 |

#### 第四阶段：事后复盘（24小时内）
1. **根因分析**：为什么漏了？（未录入/提醒被忽略/计算错误/交接遗漏）
2. **流程改进**：如何防止再次发生？
3. **系统优化**：更新预警规则、增加校验机制
4. **记录归档**：完整记录事故经过、处理过程和教训

### 时效事故分类与对策

| 事故原因 | 预防措施 | 检测手段 |
|----------|----------|----------|
| 未录入时效 | 新案件建档时自动识别时效节点 | 每周一全量扫描案件，检查是否有未录入时效 |
| 提醒被忽略 | P0 时效必须确认回执，不确认则升级通知 | 升级通知链：飞书→邮件→电话 |
| 计算错误 | 时效计算公式校验 + 双人复核 | AI 自动计算 + 律师确认 |
| 交接遗漏 | 案件交接清单必须包含全部时效 | 交接时自动生成时效清单 |
| 假日/延期误判 | 自动识别法定节假日，顺延计算 | 对接官方节假日数据 |

## 时效管理原则

1. **录入即生效**：任何时效一经录入，系统自动追踪，不依赖人工记忆
2. **宁早勿晚**：提前提醒比滞后好，多提醒比漏了好
3. **确认机制**：高优先级时效必须收到确认回执，不确认就持续提醒
4. **定期巡检**：每周一扫描全部时效清单，确保无遗漏
5. **复盘改进**：每次时效事故都要复盘，优化预警机制

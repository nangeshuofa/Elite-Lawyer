# 案件流程引擎 — 标准化办案流程

> 核心思想：把办案流程标准化、模块化，减少重复劳动，确保质量底线。

## 全流程五阶段

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  接案评估 │ →  │  深度准备 │ →  │  执行交付 │ →  │  结案复盘 │ →  │  归档沉淀 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↓              ↓              ↓              ↓              ↓
  客户画像      证据清单      庭审/谈判      经验总结      知识入库
  案件评估      法律研究      客户沟通      费用结算      案例入库
  风险告知      策略制定      文书交付      客户回访      模板更新
```

## 第一阶段：接案评估（1-2天）

### 标准动作
| 步骤 | 动作 | 输出 | 负责人 |
|------|------|------|--------|
| 1 | 听取当事人陈述 + 初步询问 | 案情摘要 | 律师+助手 |
| 2 | 初步法律分析 | 法律关系初步判断 | AI辅助+律师 |
| 3 | 利益冲突检索 | 利益冲突检索报告 | AI |
| 4 | 风险评估 + 结果预判 | 案件风险评估表 | 律师 |
| 5 | 收费方案制定 | 律师费报价 | 律师 |
| 6 | 决定是否接受委托 | 委托合同/婉拒函 | 律师 |

### 关键检查点
- □ 利益冲突已排查
- □ 风险已充分告知
- □ 收费方案已确认
- □ 委托手续已签署

## 第二阶段：深度准备（持续至开庭/交付）

### 标准动作
| 模块 | 动作 | 输出 |
|------|------|------|
| 证据管理 | 收集→分类→编号→质证准备 | 证据清单 + 质证意见 |
| 法律研究 | 法条→司法解释→类案→裁判规则 | 法律检索报告 |
| 策略制定 | 诉讼请求/抗辩思路→证据链→风险点 | 诉讼策略方案 |
| 文书起草 | 起诉状/答辩状/代理词/证据清单 | 文书初稿→定稿 |

### AI 可承担的工作
- 自动检索相关法条和司法解释
- 自动检索同类案件裁判倾向
- 自动生成证据清单（按时间/证明对象排序）
- 基于模板生成文书初稿
- 自动校对格式、法条引用、错别字

## 第三阶段：执行交付

### 庭审前准备（开庭前3天）
- □ 证据原件核对
- □ 庭审提纲准备
- □ 对方策略预判
- □ 应变方案准备
- □ 开庭提醒 + 材料清单确认

### 庭审后跟进
- □ 庭审记录整理
- □ 补充证据/代理意见提交
- □ 客户沟通 + 庭审汇报
- □ 等待判决期间的跟进

## 第四阶段：结案复盘

### 结案类型
- 胜诉结案 / 败诉结案 / 调解结案 / 撤诉结案 / 其他

### 标准复盘
| 复盘项 | 内容 |
|--------|------|
| 结果评估 | 是否达到预期？偏差在哪里？ |
| 策略复盘 | 诉讼策略是否正确？哪些可以优化？ |
| 证据复盘 | 证据链是否完整？哪些是关键证据？ |
| 客户反馈 | 客户满意度？有哪些改进空间？ |
| 经验教训 | 最大的收获/教训是什么？ |
| 可复用知识 | 有哪些可以沉淀到知识库？ |

## 第五阶段：归档沉淀

### 归档标准
- 完整卷宗：委托手续 + 证据材料 + 文书 + 庭审记录 + 裁判文书
- 电子归档：PDF 扫描件，按目录分类
- 案件标签：案由、领域、结果、关键词

### 知识沉淀（自动触发）
```
结案 → 自动提取
    ├── 裁判规则 + 法律论点 → 裁判规则库
    ├── 文书范本 → 模板库
    ├── 案例研究 + 经验教训 → 案例库
    └── 客户信息更新 → 客户管理
```

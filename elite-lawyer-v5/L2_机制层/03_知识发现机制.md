# 知识发现机制 — 越用越聪明

> 核心思想：每一次工作产出都是知识矿藏，主动挖掘→提炼→归档→复用，形成正向循环。

## 发现触发点

### 自动触发（任务完成后自动执行）
| 触发场景 | 提取目标 | 归档方向 |
|----------|----------|----------|
| 案件结案/收到判决 | 法律论点、裁判规则、证据策略、败诉教训 | 案例库 + 裁判规则 |
| 合同审核完成 | 风险点清单、标准条款、行业特殊条款 | 模板库 + 裁判规则 |
| 法律咨询完成 | 常见问题、解答模板、法律依据 | 模板库 + 法规库 |
| 庭审结束 | 质证策略、法官倾向、庭审技巧 | 案例库 + 裁判规则 |
| 文书定稿 | 优秀表述、论证结构、格式模板 | 模板库 |

### 手动触发（律师主动标注）
- "这个点很重要，记一下" → 录入知识库
- "以后类似案子可以参考这个" → 标记为标杆案例
- "这篇文章/法规有用" → 收录到对应知识库

## 知识分类体系

| 类型标识 | 名称 | 说明 | 示例 |
|----------|------|------|------|
| `[RULE]` | 裁判规则 | 法院裁判的标准和规则 | "医疗过错认定的三要素" |
| `[TPL]` | 模板范本 | 可复用的文书、合同、清单 | "民间借贷起诉状模板" |
| `[CASE]` | 案例研究 | 典型案例分析与经验 | "XX医疗损害案胜诉要点" |
| `[STRAT]` | 策略方法 | 诉讼、谈判、沟通策略 | "医疗纠纷举证策略" |
| `[KNOW]` | 知识要点 | 行业知识、法律知识点 | "病历封存的程序要求" |

## 知识卡片标准格式

```markdown
### [类型] 知识标题
- **来源**：案件名/任务名/日期
- **领域**：医疗法律/劳动法/合同法/...
- **摘要**：一句话核心结论
- **详情**：具体内容（3-5句话，可引用原文）
- **法律依据**：相关法条/司法解释
- **复用场景**：什么时候可以再次使用
- **关联**：相关案例/模板/文件路径
- **添加日期**：YYYY-MM-DD
```

## 发现流程

```
任务完成
   ↓
AI 自动扫描产出物
   ↓
┌─ 是否包含可复用知识？─┐
│                        │
是                       否
↓                        ↓
提取关键内容          正常归档
   ↓
标注知识类型
   ↓
律师审核确认（可选，高价值知识）
   ↓
写入 L4 对应知识库
   ↓
更新知识库索引
```

## 定期知识整理

| 周期 | 整理内容 | 产出 |
|------|----------|------|
| 每周 | 扫描本周新增知识，去重、分类、补充 | 知识库更新记录 |
| 每月 | 整理当月高频法律问题，形成专题知识库 | 专题知识包 |
| 每季 | 知识库健康检查：过期标记、重复合并、索引优化 | 知识库健康报告 |
| 每年 | 年度知识大盘点，识别知识盲区，规划学习方向 | 知识体系升级方案 |

## 知识复用机制

### 主动推荐（触发条件 + 推荐内容）

| 触发条件 | 自动推荐内容 | 推荐优先级 |
|----------|-------------|-----------|
| 新建案件（同领域） | 同类标杆案例 + 相关裁判规则 + 文书模板 | 高 |
| 新建案件（同客户） | 客户历史案件 + 服务记录 + 客户偏好 | 高 |
| 开始合同审核（同类型） | 同类合同常见风险点 + 标准条款 + 历史审核意见 | 中 |
| 开始法律检索 | 历史同类检索报告 + 已验证的法条列表 | 中 |
| 开庭准备 | 同类案件庭审策略 + 法官倾向 + 质证要点 | 高 |
| 客户咨询 | 历史同类咨询记录 + 标准答复模板 | 中 |
| 收到对方证据 | 同类证据的质证要点 + 反驳案例 | 高 |

### 推荐展示格式
```
💡 基于你的知识库，以下内容可能对当前任务有帮助：

【同类案例】（3个以内）
  1. [案号] 案由 — 裁判要旨一句话 — 相似度：高/中

【相关裁判规则】（3条以内）
  1. [规则名称] — 适用场景

【可用模板】
  1. [模板名称] — 匹配度：高/中

是否需要我调取详细内容？
```

### 检索调用
- 关键词检索：支持按领域、类型、关键词、标签搜索
- 案件关联：每个案件自动关联相关知识卡片
- 模板调用：写文书时自动匹配相关模板
- 智能去重：推荐时自动过滤已使用/已确认不相关的知识

## 知识库质量标准

1. **来源可查**：每条知识都有明确来源，可追溯验证
2. **准确可靠**：法律依据准确，案例真实，不编造
3. **结构清晰**：标准化卡片格式，易读易检索
4. **持续更新**：法规更新时同步更新知识库
5. **去重合并**：相似内容定期合并，避免冗余

## 知识价值评估

- 高频调用 = 高价值 → 重点维护
- 低频但关键 = 战略价值 → 保留但不优先
- 长期不用 = 低价值 → 定期清理或降级
